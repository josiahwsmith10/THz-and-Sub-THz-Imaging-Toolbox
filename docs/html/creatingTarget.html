Creating a Target Scene and Simulating the Radar Beat Signal from the Target Scene

Using Interactive GUI

        Under the Target Setup tab, the user can modify many parameters relating to the various methods of importing targets and generating target scenes.

o   Under the Options panel, the user can select which types of targets to include: Table of Targets, PNG file, STL file

  Use GPU check box indicates if the program should use the GPU to compute the beat signal

  Path Loss check box indicates if the program should include the round-trip path loss in the simulation

o   Under the Compute Beat Signal panel, the user can simulate the beat signal for the current target scene, shown in the figure

  If the Compute Beat Signal box is pressed, the beat signal is computed and the Beat Signal Computed lamp in the bottom right corner is updated accordingly

 

        Table of Targets panel

o     The user can specify the exact position and reflectivity of each point in the scene, treated as ideal back-scatterers

o   The user can add or remove targets, removing the bottom most element of the table

 

        Import PNG Image panel

o   The user can specify the name of the image they wish to import (including its extension)

o   The imported image is treated as a 2-D image contained in an X-Y plane

o   X-Pixel Size (m) scales the imported image equating 1 x-pixel of the PNG file to the value of the field

o   Y-Pixel Size (m) scales the imported image equating 1 y-pixel of the PNG file to the value of the field

o   X Offset (m) is the offset of the PNG from the origin in the x-direction

o   Y Offset (m) is the offset of the PNG from the origin in the y-direction

o   Z Offset (m) is the offset of the PNG from the origin in the z-direction

o   Reflectivity is the reflectivity of the PNG assigned to every point of the image

o   Downsample Factor specifies the factor by which to downsample the PNG file to reduce the number of target voxels and thus the computation time of the beat signal

o   Num Targets shows the number of voxels making up the target created by the PNG file

 

        Import STL File panel

o   The user can specify the name of the STL file they wish to import (including its extension)

o   Z Crop (m) crops all points past the value of the field. This can be helpful when it is not necessary to simulate the beat signal from the entire target, but rather only the front of the target

o   X Offset (m) is the offset of the PNG from the origin in the x-direction

o   Y Offset (m) is the offset of the PNG from the origin in the y-direction

o   Z Offset (m) is the offset of the PNG from the origin in the z-direction

o   Reflectivity is the reflectivity of the STL target assigned to every point of the target

o   Downsample Factor specifies the factor by which to downsample the STL file to reduce the number of target voxels and thus the computation time of the beat signal

o   Num Targets shows the number of voxels making up the target created by the STL file

 

        Generate Random Points panel

o   The user can generate points randomly given the parameters in this panel. This can be a useful strategy for demonstration and for dataset generation as discussed later

 

        As the parameters of the target scenario are changed by the user (or the parameters of the SAR scenario or antenna array), the figure is automatically updated show the SAR array and target

o   It is important to note that the array is synthesized in TDM MIMO-SAR mode, meaning a signal transmitted from a single transmitter is not captured by all the receivers. Rather, only the receivers making up the antenna array at that given step in the scan.

Saving and Loading Targets

Using the Interactive GUI

        The current target can be saved by pressing the Save button under the Save/Load Target panel

o   If the name given in the field to the right of the save button refers to an existing file under /application/saved/targets/, the user is prompted whether they would like to overwrite the file

        The user can load previously saved targets by pressing the Load button

o   If the file does not exist under /application/saved/targets/, the user is prompted, and the load fails
